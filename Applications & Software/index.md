# Applications & Software

## Build

**Understand the Landscape:** Get a clear picture of all the software and applications used across the organization.

**Establish Governance:** Create a framework for how software is selected, deployed, managed, and retired.

**Define Key Roles:** Identify who is responsible for application ownership, support, and development.

**Create a Central Catalog:** Develop a single source of truth that lists all approved applications and provides essential information about them.

## Manage

**Oversee the Portfolio:** Regularly review the entire application portfolio to identify redundancies, gaps, and opportunities for improvement.

**Facilitate Collaboration:** Ensure that teams responsible for development, support, and infrastructure are working together effectively.

**Track Performance and Value:** Monitor how well applications are meeting business needs and providing a return on investment.

**Communicate Strategy:** Keep the organization informed about the overall application strategy and any upcoming changes.

## Maturity

- [ ] Do you have a high-level view of your entire application portfolio?
- [ ] Is there a clear process for requesting a new application?
- [ ] Do you know which applications are critical for business operations?
- [ ] Is there a strategy for managing the entire application lifecycle?
