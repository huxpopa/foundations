# Database Services

## Build

**List Your Databases:** Create a complete inventory of all databases and the applications that depend on them.

**Assign Ownership:** Designate a specific person or team accountable for the administration and health of each database.

**Define Service Levels:** Clearly document expectations for database uptime, performance, and backup frequency.

**Establish Standard Procedures:** Create and document processes for:
- **Backup and Recovery:** How data is backed up and how it can be restored.
- **Security:** How access is controlled and data is protected.
- **Monitoring:** How database health and performance are tracked.
- **Changes:** How modifications to the database are requested, approved, and implemented.

## Manage

**Conduct Regular Reviews:** Hold periodic meetings with database owners and application teams to review performance and address any issues.

**Monitor Key Metrics:** Track metrics on database performance, storage capacity, and successful backups.

**Identify and Track Improvements:** Continuously look for opportunities to optimize, secure, and improve database services.

**Communicate Changes:** Ensure all stakeholders are notified of any planned maintenance or changes to the database environment.

## Maturity

- [ ] Do you have a centralized list of all your databases?
- [ ] Is every database's owner clearly identified?
- [ ] Do you have a documented recovery plan for your critical databases?
- [ ] Are you actively monitoring database performance?
